- **框架**: Magic-API 2.1.1_v3 + Spring Boot 2.7.4
- **脚本引擎**: JSR223
- **默认数据库**: MongoDB
- **应用ID**: 15e5b882231520462e3435fded48cb3c

### 1.2 强制性规则

1. ✅ 优先参考项目中的实际案例（`api/` 目录）
2. ✅ 严格遵循 JSR223 语法规范
3. ✅ MongoDB projection 中不能同时使用包含(1)和排除(0)字段
4. ✅ 必须使用 `response.json()` 返回结果

10. ✅ **⭐⭐⭐ 用户提供的是表单code，不是表单ID**：直接使用用户提供的code

## 2. JSR223语法规范

### 2.1 循环语法 ⭐⭐⭐ 最重要

#### 🚨🚨🚨 数组 vs Map/Object 遍历语法完全不同！

**这是最容易犯错的地方，必须严格区分！**

| 数据类型 | 正确语法 | 说明 |
|---------|---------|------|
| **数组 Array** | `for (item in array)` | 单变量，拿到的是元素 |
| **数组 Array（带索引）** | `for (i, item in array)` | 双变量，i是索引，item是元素 |
| **Map/Object** | `for (key, value in map)` | **必须双变量**，key是键，value是值 |

#### ✅ 数组遍历 - 正确写法

```javascript
// 数组遍历 - 只需要元素（最常用）
var list = [1, 2, 3];
for (item in list) {
    log.info("值: " + item);  // 输出 1, 2, 3
}

// 数组遍历 - 带索引
for (i, item in list) {
    log.info("索引: " + i + ", 值: " + item);
}

// 从数据库查询的结果是数组
var materialList = mongo.database().collection(formId).find({}).list();
for (item in materialList) {
    log.info(item.materialCode);  // ✅ 正确
}
```

#### ✅ Map/Object遍历 - 正确写法

```javascript
// Map遍历 - 必须用双变量！
var demandMap = {};
demandMap["M001"] = { quantity: 100 };
demandMap["M002"] = { quantity: 200 };

// ✅ 正确：双变量遍历Map
for (code, data in demandMap) {
    log.info("物料: " + code + ", 数量: " + data.quantity);
}

// ✅ 如果只需要keys，也必须用双变量，忽略value即可
var keys = [];
for (key, value in demandMap) {
    keys.push(key);
}

// ✅ 或者用 .keySet() 转成数组再遍历
var keys = demandMap.keySet();
for (key in keys) {
    var value = demandMap[key];
}
```

#### ❌ Map遍历 - 致命错误！

```javascript
// ❌❌❌ 严重错误：用数组语法遍历Map
var demandMap = { "M001": {qty: 100}, "M002": {qty: 200} };

// ❌ 错误！这样写code拿到的不是key！
for (code in demandMap) {
    materialCodes.push(code);  // 结果不正确！
}

// ✅ 正确写法
var materialCodes = [];
for (code, data in demandMap) {  // ← 必须双变量
    materialCodes.push(code);
}
```

#### 🔍 如何判断是数组还是Map？

| 来源 | 类型 | 遍历语法 |
|------|------|---------|
| `mongo.find().list()` | 数组 | `for (item in list)` |
| `body.products` | 数组 | `for (product in products)` |
| `var map = {}; map[key] = value;` | Map | `for (key, value in map)` |
| `getFormIdByCodes([...])` 返回值 | Map | `for (code, formId in formMap)` |
| 函数内部构建的 `xxxMap` | Map | `for (key, value in xxxMap)` |

var errorSummary = [];

## 7. 常见错误速查

### 7.1 语法错误对照表

| 错误写法 | 正确写法 | 严重程度 |
|---------|---------|---------|
| `for (code in map)` 遍历Map | `for (code, data in map)` | 🚨 致命 |
| `for (var i = 0; i < arr.length; i++)` | `for (i, item in arr)` | 🚨 致命 |
| `new BigDecimal(value)` | `value.asDecimal()` | ❌ 错误 |
| `parseInt(value)` | `value.asInt(0)` | ❌ 错误 |
| `const x = 1` | `var x = 1` | ❌ 错误 |
| `let x = 1` | `var x = 1` | ❌ 错误 |
| `arr.forEach(...)` | `for (item in arr)` | ❌ 错误 |
| 硬编码表单ID | `getFormIdByCodes(["表单编码"])` | ❌ 错误 |
| `if (x) return;` | `if (x) { return; }` | ❌ 错误 |
| `if (x) y = 1; else y = 2;` | `if (x) { y = 1; } else { y = 2; }` | ❌ 错误 |
| `a + b` (数值计算) | `a.asDecimal().add(b.asDecimal())` | ⚠️ 精度问题 |
| `a * b` (数值计算) | `a.asDecimal().multiply(b.asDecimal())` | ⚠️ 精度问题 |

### 7.2 致命错误案例：Map遍历用错语法

```javascript
// ❌ 这个错误会导致数据查不到、逻辑完全错误！
var demandMap = { "M001": {...}, "M002": {...} };
var materialCodes = [];
for (code in demandMap) {  // ← 致命错误！
    materialCodes.push(code);
}
// 结果：materialCodes 可能是空的或数据不对，后续查询全部失败！

// ✅ 正确写法
for (code, data in demandMap) {
    materialCodes.push(code);
}
```
