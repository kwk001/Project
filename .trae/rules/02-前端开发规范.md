## 1. 概述

### 1.1 什么是 JSX 低代码开发模式

JSX 低代码开发模式是一种基于 React 类组件的编程式开发方式，核心特点是：

- **编程式 UI**：通过编写 JSX 代码构建界面，而非配置 JSON
- **renderJSX 方法**：所有 UI 渲染逻辑集中在 `renderJSX()` 方法中
- **Antd 组件库**：强制使用 Ant Design 组件库
- **服务化架构**：通过 DataService 访问后端服务
- **类组件模式**：使用 React 类组件，不使用函数组件

### 1.2 与传统低代码的区别

| 特性 | 传统低代码 (KaiwuFlex) | JSX 低代码 |
|------|----------------------|-----------|
| UI 构建 | JSON 配置 | JSX 编程 |
| 组件库 | KaiwuFlex 组件 | Antd 原生组件 |
| 灵活性 | 配置受限 | 完全自由 |
| 学习曲线 | 低 | 中等 |
| 适用场景 | 标准 CRUD | 复杂交互 |

### 1.3 适用场景

✅ **适合使用 JSX 低代码**：
- 复杂的自定义交互逻辑
- 需要精细控制 UI 布局
- 动态渲染复杂组件
- 需要使用 Antd 高级特性

❌ **不适合，建议用 KaiwuFlex**：
- 简单的 CRUD 列表页
- 标准表单页面
- 快速原型开发

### 1.4 重要提示 ⚠️

**🚨 assets.json 文件管理规则：**

- ❌ **AI 不要修改 assets.json 文件**
- ✅ **assets.json 需要手动添加和维护**
- ✅ **从现有页面复制 assets.json 作为模板**
- ✅ **确保包含必需的依赖包（如 DayjsLocale、KaiwuMesModule 等）**

**原因：**
- assets.json 包含复杂的 CDN 依赖配置
- 不同项目的依赖版本可能不同
- 手动管理可以避免依赖冲突和版本问题
- AI 修改可能导致依赖配置混乱

## 2. 核心原则

### 2.1 强制要求

1. **开发前必须使用MCP工具查看表单结构** - 确保字段名一致
2. **JSX模式**：所有UI在`renderJSX()`中，从`window.antd`获取组件
3. **KaiwuFlex模式**：JSON配置，适合标准CRUD
4. **样式优先CSS类名** - 避免行内样式，必须保留三个基础初始化样式
5. **布局使用Flex/Grid** - 禁止float/table布局
6. **先获取formId再查询** - 不要硬编码表单ID
7. **代码可读性优先** - 简单赋值、短条件可以写成一行；JSX 属性超过 2 个、函数体有多条语句、嵌套结构时，必须换行展开，不允许为了紧凑而牺牲可读性
8. **大文件分段写入** - 预估内容较多的文件，必须分段写入，每段控制在 300～500 行以内；禁止一次性写入超大文件，避免截断或写入失败

### 2.2 开发流程

1. 使用MCP工具查看表单结构
2. 明确需求和业务规则
3. 选择开发模式（JSX或KaiwuFlex）
4. 复制模板文件
5. 实现功能代码
6. 调整样式
7. 测试验证

### 2.3 文件管理规则

| 文件 | AI修改规则 |
|------|-----------|
| `index.jsx` | ✅ 可以直接修改 |
| `index.css` | ✅ 可以修改，但必须保留三个基础样式 |
| `schema.json` | ⚠️ 修改前必须询问用户 |
| `assets.json` | ❌ 绝对不要修改（手动管理）推荐去低代码编辑器中配置 |

## 5. 数据服务使用

### 5.1 查询条件说明

```javascript
{
  field: "字段名",
  conditionOperator: "操作符",
  conditionValues: [值]   // 即使单个值也要用数组
}
```

**常用操作符**：

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `eq` | 等于 | `{ field: 'status', conditionOperator: 'eq', conditionValues: ['active'] }` |
| `ne` | 不等于 | `{ field: 'status', conditionOperator: 'ne', conditionValues: ['deleted'] }` |
| `like` | 模糊查询 | `{ field: 'name', conditionOperator: 'like', conditionValues: ['产品'] }` |
| `gt` | 大于 | `{ field: 'quantity', conditionOperator: 'gt', conditionValues: ['0'] }` |
| `gte` | 大于等于 | `{ field: 'quantity', conditionOperator: 'gte', conditionValues: ['10'] }` |
| `lt` | 小于 | `{ field: 'price', conditionOperator: 'lt', conditionValues: ['100'] }` |
| `lte` | 小于等于 | `{ field: 'price', conditionOperator: 'lte', conditionValues: ['1000'] }` |
| `eqa` | IN查询 | `{ field: 'status', conditionOperator: 'eqa', conditionValues: ['active', 'pending'] }` |
| `between` | 区间查询 | `{ field: 'createTime', conditionOperator: 'between', conditionValues: ['2024-01-01', '2024-12-31'] }` |

### 5.2 标准查询模式

```javascript
async loadData() {
  await this.setStatePromise({ loading: true });
  try {
    const formId = await this.formService.getFormId('your_form_code');
    const res = await this.dataSourceMap.queryFormData.load({
      formId,
      conditionFilter: {
        conditionType: "and",
        conditions: [{ field: 'status', conditionOperator: 'eq', conditionValues: ['active'] }]
      },
      page: { current: this.state.pageNo, size: this.state.pageSize },
      sorts: []
    });
    if (res.code === 200 && res?.result?.records?.length > 0) {
      await this.setStatePromise({ tableData: res.result.records, total: res.result.total });
    }
  } catch (error) {
    const { message } = window.antd;
    message.error('查询失败');
  } finally {
    await this.setStatePromise({ loading: false });
  }
}
```

**返回数据结构**：

```javascript
// ⭐ 数据在 result 中，不是 data
{ code: 200, result: { records: [...], total: 10, current: 1, size: 10 } }
```

### 5.4 schema.json数据源配置

#### 5.4.1 mainApi - 平台标准API

**常用接口**：

| 接口路径 | 方法 | 说明 |
|---------|------|------|
| `/mainApi/formEngine/formData/query` | POST | 查询 |
| `/mainApi/formEngine/formData/add` | POST | 新增 |
| `/mainApi/formEngine/formData/update` | PUT | 更新 |
| `/mainApi/formEngine/formData/delete` | DELETE | 删除 |
| `/mainApi/formEngine/form/getIdByCode` | POST | 根据编码获取表单ID |
| `/mainApi/identity/user/query/current` | GET | 获取当前用户 |

schema.json 配置格式：
```json
{ "id": "queryFormData", "type": "fetch", "uri": "/mainApi/formEngine/formData/query", "method": "POST",
  "shouldFetch": { "type": "JSFunction", "value": "function() { return true; }" } }
```

#### 5.4.2 kaiwuApi - 客户自定义API

**用途**：调用通过 Magic API 开发的业务定制接口，路径格式：`/kaiwuApi/${customerId}/接口地址`

**适用场景**：复杂业务逻辑、多表关联查询、数据聚合统计、第三方系统集成

**配置示例**（uri 必须用 JSExpression 动态获取 customerId）：

```json
{ "type": "fetch", "id": "customBusinessApi",
  "uri": { "type": "JSExpression", "value": "`/kaiwuApi/${this.state.currentUser.customerId}/mes/production/getDetail`" },
  "shouldFetch": { "type": "JSFunction", "value": "function() { return true; }" } }
```

**两种API对比**：

| 特性 | mainApi | kaiwuApi |
|------|---------|----------|
| 用途 | 平台标准CRUD | 业务定制逻辑 |
| 路径 | 固定字符串 | JSExpression（含customerId） |
| 开发方式 | 平台内置 | Magic API 开发 |

## 8. 组件拆分与最佳实践

### 8.1 renderJSX拆分

复杂页面拆分为多个渲染方法，主方法只做组合：

```javascript
renderSearchArea() {
  const { Input, Button, Space, Select } = window.antd;
  return (
    <div className="search-row">
      <Input.Search placeholder="搜索" onSearch={() => this.handleSearch()} className="search-input" />
      <Select placeholder="选择状态" allowClear onChange={(val) => this.setState({ searchStatus: val || '' })}>
        <Select.Option value="active">启用</Select.Option>
        <Select.Option value="inactive">禁用</Select.Option>
      </Select>
      <Space>
        <Button type="primary" onClick={() => this.handleSearch()}>查询</Button>
        <Button type="primary" onClick={() => this.handleAdd()}>新增</Button>
      </Space>
    </div>
  );
}

renderTable() {
  const { Table, Pagination } = window.antd;
  return (
    <>
      <div className="table-scroll-wrap">
        <Table dataSource={this.state.tableData} columns={this.getColumns()} loading={this.state.loading} rowKey="_id" pagination={false} />
      </div>
      <div className="pagination-bar">
        <Pagination current={this.state.pageNo} pageSize={this.state.pageSize} total={this.state.total}
          showSizeChanger showQuickJumper showTotal={(t) => `共 ${t} 条`}
          onChange={(page, size) => this.handlePageChange(page, size)} />
      </div>
    </>
  );
}

renderModal() {
  const { Modal, Form, Input, Select } = window.antd;
  return (
    <Modal title={this.state.editRecord ? '编辑' : '新增'} open={this.state.modalVisible}
      onOk={() => this.handleModalOk()} onCancel={() => this.setState({ modalVisible: false })}
      confirmLoading={this.state.modalLoading}>
      <Form ref={(ref) => this.formRef = ref} layout="vertical">
        <Form.Item label="名称" name="name" rules={[{ required: true, message: '请输入名称' }]}><Input /></Form.Item>
        <Form.Item label="状态" name="status">
          <Select><Select.Option value="active">启用</Select.Option></Select>
        </Form.Item>
      </Form>
    </Modal>
  );
}

renderJSX() {
  const { ConfigProvider, Card } = window.antd;
  return (
    <ConfigProvider theme={window.antdTheme} locale={this.getLocale()}>
      <div className="page-container">
        <Card className="search-card">{this.renderSearchArea()}</Card>
        <Card className="table-card" bodyStyle={{ flex: 1, display: 'flex', flexDirection: 'column', overflow: 'hidden' }}>
          {this.renderTable()}
        </Card>
        {this.renderModal()}
      </div>
    </ConfigProvider>
  );
}
```

## 9. 性能优化

### 9.1 避免重复查询 / 内存管理 / loading规范

```javascript
// 避免重复查询：已有数据则跳过
```

## 10. 错误处理规范

```javascript
// 统一错误处理
handleError(error, defaultMessage = '操作失败') {
  const { message } = window.antd;
  message.error(error?.response?.data?.message || error?.message || defaultMessage);
}

// 表单验证
async handleModalOk() {
  if (!this.formRef) return;
  try {
    const values = await this.formRef.validateFields();
    await this.submitData(values);
  } catch (error) {
    if (error.errorFields) return; // Antd自动显示验证错误
    this.handleError(error, '提交失败');
  }
}

// 删除确认
handleDelete(record) {
  const { Modal, message } = window.antd;
  Modal.confirm({
    title: '确认删除', content: `确定要删除"${record.name}"吗？`, okType: 'danger',
    onOk: async () => {
      try {
        await this.dataSourceMap.deleteFormData.load({ formId: this.state.formMap.your_form_code, data: { _id: [record._id] } });
        message.success('删除成功'); this.loadData();
      } catch { message.error('删除失败'); }
    }
  });
}

// 批量删除
async handleBatchDelete() {
  const { selectedRowKeys } = this.state;
  const { Modal, message } = window.antd;
  if (!selectedRowKeys.length) { message.warning('请先选择数据'); return; }
  Modal.confirm({
    title: `批量删除 ${selectedRowKeys.length} 条`, okType: 'danger',
    onOk: async () => {
      try {
        await this.dataSourceMap.deleteFormData.load({ formId: this.state.formMap.your_form_code, data: { _id: selectedRowKeys } });
        message.success('删除成功'); this.setState({ selectedRowKeys: [] }); this.loadData();
      } catch { message.error('批量删除失败'); }
    }
  });
}
```

## 11. 常见错误速查

### 11.1 错误对照表

| 错误 | 正确 | 严重程度 |
|------|------|---------|
| `import { Button } from 'antd'` | `const {Button} = window.antd` | 🚨 致命 |
| `render() { return <div/> }` | `renderJSX() { return <div/> }` | 🚨 致命 |
| 不用`ConfigProvider`包裹 | 必须用`ConfigProvider`包裹 | 🚨 致命 |
| `export default LowcodeComponent` | 低代码平台不需要导出，删除此行 | 🚨 致命 |
| 直接修改`this.state.x = y` | 使用`this.setState({x: y})` | ❌ 错误 |
| 查询参数直接传`conditions` | 包装在`conditionFilter`中 | ❌ 错误 |
| 数据在`result.data.records` | 数据在`result.result.records` | ❌ 错误 |
| 未获取formId就查询 | 先`getFormId()`再查询 | ❌ 错误 |
| `onClick={this.handleClick()}` | `onClick={() => this.handleClick()}` | ❌ 错误 |
| 大量行内样式 | 使用CSS类名 | ⚠️ 不推荐 |
| 使用float布局 | 使用Flex/Grid | ⚠️ 不推荐 |
| renderJSX返回多个根元素 | 用单个div包裹 | ❌ 错误 |
| 异步操作不处理loading | 必须有loading状态 | ⚠️ 不推荐 |

### 11.2 错误示例对照

```javascript
// ❌ 错误1：import导入Antd
import { Button } from 'antd';
// ✅ 正确
const { Button } = window.antd;

// ❌ 错误2：使用render()而非renderJSX()，且未包ConfigProvider
render() { return <div />; }
// ✅ 正确
renderJSX() {
  const { ConfigProvider } = window.antd;
  return <ConfigProvider theme={window.antdTheme} locale={this.getLocale()}><div /></ConfigProvider>;
}

// ❌ 错误3：事件绑定（直接引用/直接调用）
<Button onClick={this.handleClick}>  // this丢失
<Button onClick={this.handleClick()}>  // 立即执行
// ✅ 正确
<Button onClick={() => this.handleClick(record)}>

// ❌ 错误4：直接修改state
this.state.loading = true;
// ✅ 正确
this.setState({ loading: true });

// ❌ 错误5：未获取formId就查询（formMap初始值是空字符串）
this.dataSourceMap.queryFormData.load({ formId: this.state.formMap.your_form_code });
// ✅ 正确
const formId = await this.formService.getFormId('your_form_code');
this.dataSourceMap.queryFormData.load({ formId });

// ❌ 错误6：异步操作不处理loading（报错时loading永远不重置）
async loadData() { const data = await this.fetchData(); this.setState({ tableData: data, loading: false }); }
// ✅ 正确
async loadData() {
  this.setState({ loading: true });
  try { const data = await this.fetchData(); this.setState({ tableData: data }); }
  catch (e) { message.error('加载失败'); }
  finally { this.setState({ loading: false }); }
}
```
