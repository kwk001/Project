# UI/UX 设计指南

## 1. 设计系统基础

### 1.1 什么是设计系统
设计系统是一套标准化的设计规范，包括色彩、字体、间距、组件等，确保产品视觉一致性和开发效率。

### 1.2 设计系统价值
- **一致性**：统一视觉风格，提升品牌认知
- **效率**：复用组件，减少重复设计
- **协作**：设计师与开发使用同一语言
- **可维护**：易于更新和扩展

## 2. 色彩系统

### 2.1 主色调

| 颜色 | 色值 | 用途 |
|------|------|------|
| 主色 | `#1890ff` | 主要按钮、链接、高亮 |
| 主色-浅色 | `#40a9ff` | 悬浮状态 |
| 主色-深色 | `#096dd9` | 点击状态 |

### 2.2 功能色

| 颜色 | 色值 | 用途 |
|------|------|------|
| 成功 | `#52c41a` | 成功提示、通过状态 |
| 警告 | `#faad14` | 警告提示、待处理 |
| 错误 | `#f5222d` | 错误提示、失败状态 |
| 信息 | `#1890ff` | 信息提示 |

### 2.3 中性色

| 颜色 | 色值 | 用途 |
|------|------|------|
| 标题文字 | `#262626` | 一级标题 |
| 正文文字 | `#595959` | 正文内容 |
| 辅助文字 | `#8c8c8c` | 辅助说明 |
| 禁用文字 | `#bfbfbf` | 禁用状态 |
| 边框 | `#d9d9d9` | 边框、分割线 |
| 背景 | `#f5f5f5` | 页面背景 |
| 卡片背景 | `#ffffff` | 卡片、弹窗 |

### 2.4 色彩使用原则

**1. 60-30-10 原则**
- 60% 中性色（背景、文字）
- 30% 主色（主要元素）
- 10% 强调色（按钮、标签）

**2. 对比度要求**
- 正文与背景对比度 ≥ 4.5:1
- 大文字与背景对比度 ≥ 3:1

**3. 色彩语义**
- 同一颜色表达相同含义
- 红色 = 危险/删除
- 绿色 = 成功/通过
- 黄色 = 警告/待处理

## 3. 字体系统

### 3.1 字体选择

**中文字体：**
- 首选：系统默认字体（-apple-system, "Segoe UI", "PingFang SC", "Microsoft YaHei"）
- 备用："Helvetica Neue", Arial, sans-serif

**英文字体：**
- 首选：-apple-system, BlinkMacSystemFont
- 备用："Segoe UI", Roboto, "Helvetica Neue", Arial

### 3.2 字号规范

| 级别 | 字号 | 行高 | 字重 | 用途 |
|------|------|------|------|------|
| H1 | 38px | 1.4 | 600 | 页面大标题 |
| H2 | 30px | 1.4 | 600 | 区块标题 |
| H3 | 24px | 1.4 | 600 | 小标题 |
| H4 | 20px | 1.4 | 600 | 卡片标题 |
| H5 | 16px | 1.4 | 600 | 列表标题 |
| 正文 | 14px | 1.6 | 400 | 正文内容 |
| 辅助 | 12px | 1.5 | 400 | 辅助说明 |

### 3.3 字体使用原则

**1. 层次清晰**
- 标题醒目，正文易读
- 通过字号、字重区分层次
- 避免使用过多字号

**2. 行高舒适**
- 标题行高：1.2-1.4
- 正文行高：1.5-1.8
- 中文行高略高于英文

**3. 字数控制**
- 单行文字 25-35 个中文字符
- 段落宽度不超过 700px

## 4. 间距系统

### 4.1 8px 网格系统

所有间距基于 8px 的倍数：

| Token | 值 | 用途 |
|-------|-----|------|
| xs | 4px | 图标与文字间距 |
| sm | 8px | 紧凑间距 |
| md | 16px | 默认间距 |
| lg | 24px | 区块间距 |
| xl | 32px | 大区块间距 |
| xxl | 48px | 页面间距 |

### 4.2 间距类型

**内边距（Padding）：**
- 小卡片：16px
- 中卡片：24px
- 大卡片：32px

**外边距（Margin）：**
- 元素间：16px
- 区块间：24-32px
- 页面边距：24-48px

**间距原则：**
- 相关元素间距小
- 不相关元素间距大
- 保持一致的间距节奏

## 5. 布局系统

### 5.1 响应式断点

| 断点 | 宽度 | 设备 |
|------|------|------|
| xs | < 576px | 手机 |
| sm | ≥ 576px | 大手机 |
| md | ≥ 768px | 平板 |
| lg | ≥ 992px | 笔记本 |
| xl | ≥ 1200px | 桌面 |
| xxl | ≥ 1600px | 大屏 |

### 5.2 栅格系统

**24 列栅格：**
- 总宽度：100%
- 列数：24
- 列间距（Gutter）：16px（默认）
- 边距（Margin）：24px（默认）

**使用示例：**
```
| 边距 | 列 | 间距 | 列 | 间距 | 列 | 边距 |
| 24px | 8  | 16px | 8  | 16px | 8  | 24px |
```

## 6. 组件规范

### 6.1 按钮

**按钮类型：**

| 类型 | 样式 | 用途 |
|------|------|------|
| 主按钮 | 实心主色 | 主要操作 |
| 次按钮 | 边框主色 | 次要操作 |
| 文字按钮 | 文字主色 | 弱操作 |
| 链接按钮 | 文字链接 | 导航 |
| 危险按钮 | 实心红色 | 删除等危险操作 |

**按钮尺寸：**

| 尺寸 | 高度 | 内边距 | 用途 |
|------|------|--------|------|
| 大 | 40px | 16px 24px | 重要操作 |
| 中 | 32px | 8px 16px | 默认 |
| 小 | 24px | 4px 8px | 紧凑空间 |

**按钮状态：**
- 默认 → 悬浮（颜色加深） → 点击（颜色更深）
- 加载中（Loading）
- 禁用（Disabled，灰色）

### 6.2 表单

**输入框：**
- 高度：32px（默认）
- 边框：1px solid #d9d9d9
- 圆角：4px
- 内边距：8px 12px
- 聚焦：边框变为主色

**下拉选择：**
- 同输入框尺寸
- 右侧箭头图标
- 下拉面板：白色背景 + 阴影

**单选/复选：**
- 尺寸：16px
- 选中：主色填充
- 未选中：灰色边框

**表单布局：**
- 标签右对齐或顶对齐
- 标签宽度：80-120px
- 输入框宽度：根据内容定，通常 200-400px
- 错误提示：红色文字，位于输入框下方

### 6.3 表格

**表格结构：**
- 表头：灰色背景 #fafafa，加粗
- 行高：48px（默认）
- 边框：1px solid #f0f0f0
- 行悬浮：背景 #fafafa

**列宽设置：**
- 序号：60px
- 名称：自适应或固定
- 日期：120px
- 状态：100px
- 操作：150px

**分页：**
- 位于表格下方右侧
- 默认每页 10/20/50 条
- 显示总条数

### 6.4 卡片

**卡片结构：**
```
┌─────────────────────────────────┐
│ 标题                    [操作]  │  ← 头部（可选）
├─────────────────────────────────┤
│                                 │
│         内容区域                 │  ← 主体
│                                 │
├─────────────────────────────────┤
│ [取消]              [确定]      │  ← 底部（可选）
└─────────────────────────────────┘
```

**卡片样式：**
- 背景：白色
- 圆角：4px
- 阴影：0 2px 8px rgba(0,0,0,0.08)
- 内边距：24px

### 6.5 弹窗/抽屉

**弹窗（Modal）：**
- 宽度：400-800px（根据内容）
- 遮罩：黑色 50% 透明度
- 圆角：4px
- 头部：标题 + 关闭按钮
- 底部：操作按钮（右对齐）

**抽屉（Drawer）：**
- 宽度：400-600px
- 从右侧滑入
- 遮罩：黑色 30% 透明度

## 7. 交互规范

### 7.1 反馈机制

**即时反馈：**
- 按钮点击：视觉变化（颜色加深）
- 表单提交：Loading 状态
- 操作成功：Message 提示
- 操作失败：Message 提示 + 错误原因

**反馈组件：**
- **Message**：顶部居中，3秒自动消失
- **Notification**：右上角，可手动关闭
- **Modal**：重要确认，必须用户响应
- **Progress**：长时间操作，显示进度

### 7.2 状态提示

| 状态 | 颜色 | 图标 | 用途 |
|------|------|------|------|
| 成功 | 绿色 | ✓ | 操作成功 |
| 警告 | 黄色 | ! | 需要注意 |
| 错误 | 红色 | × | 操作失败 |
| 信息 | 蓝色 | i | 提示信息 |
| 加载中 | 灰色 | ⟳ | 等待中 |

### 7.3 防错设计

**1. 确认对话框**
- 删除操作必须确认
- 批量操作确认影响范围
- 重要设置变更确认

**2. 默认值**
- 表单提供合理默认值
- 减少用户输入

**3. 输入校验**
- 实时校验（失去焦点时）
- 明确错误提示
- 提供正确示例

## 8. 响应式设计

### 8.1 断点适配

**移动端（< 768px）：**
- 侧边栏收起为汉堡菜单
- 表格横向滚动
- 表单单列布局
- 卡片单列排列

**平板（768px - 992px）：**
- 侧边栏可折叠
- 表格适当收缩列
- 表单双列布局
- 卡片双列排列

**桌面（> 992px）：**
- 完整布局
- 表格展示所有列
- 表单多列布局
- 卡片多列排列

### 8.2 移动端适配要点

**1. 触摸优化**
- 按钮最小 44px
- 列表项高度 56px+
- 增大点击区域

**2. 输入优化**
- 数字输入调用数字键盘
- 日期输入调用日期选择器
- 自动聚焦第一个输入框

**3. 性能优化**
- 图片懒加载
- 虚拟滚动（长列表）
- 减少动画
