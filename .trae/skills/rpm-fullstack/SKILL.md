# 研发项目全生命周期管理 (RPM Fullstack)

## 概述

`rpm-fullstack` 是一个综合性项目管理 Skill，专为软件研发团队设计，覆盖项目从需求分析到上线发布的完整生命周期。

## 核心能力矩阵

| 能力域 | 功能点 | 自动化程度 |
|--------|--------|-----------|
| **计划管理** | 智能排期、优先级调度、延期自适应 | 高 |
| **进度跟踪** | 多维度视图、代码进度同步、实时可视化 | 高 |
| **风险管理** | 风险识别、预报、升级、闭环 | 中-高 |
| **变更管理** | 变更请求、审批流程、影响分析、计划调整 | 中 |
| **资源管理** | 负载监控、瓶颈识别、调配建议 | 中 |
| **质量管理** | 缺陷跟踪、质量门禁、指标分析 | 中 |
| **里程碑管理** | 阶段评审、自动升级、交付物管控 | 中 |
| **复盘沉淀** | 数据汇总、报告生成、经验沉淀 | 高 |

## 使用场景

### 场景1：新项目启动
```
用户：启动一个新项目，团队10人，周期3个月，敏捷开发模式

Skill动作：
1. 加载敏捷项目模板
2. 生成完整项目计划（WBS + 甘特图）
3. 预设里程碑（需求冻结、设计评审、代码冻结、UAT、上线）
4. 初始化风险/资源/缺陷跟踪表
5. 生成各角色视图配置
```

### 场景2：每日进度同步
```
用户：更新今日任务进度

Skill动作：
1. 接收任务状态更新
2. 自动计算整体进度
3. 检查延期任务并预报风险
4. 更新关键路径
5. 生成今日项目健康报告
```

### 场景3：需求变更
```
用户：提交需求变更请求

Skill动作：
1. 记录变更请求详情
2. 评估影响范围（任务、排期、资源）
3. 触发审批流程
4. 审批通过后自动调整计划
5. 通知所有关联方
```

### 场景4：里程碑评审
```
用户：到达设计评审里程碑

Skill动作：
1. 检查里程碑准入条件
2. 汇总相关交付物状态
3. 生成评审报告
4. 记录评审结果
5. 如延期则触发升级汇报
```

### 场景5：项目复盘
```
用户：项目结束，生成复盘报告

Skill动作：
1. 汇总项目全周期数据
2. 分析变更记录、风险问题、质量指标
3. 生成复盘报告（成果、问题、改进建议）
4. 沉淀经验到知识库
```

## 角色视图配置

### 管理层视图
- 关注：整体进度、里程碑状态、风险等级、资源健康度
- 展示：燃尽图、里程碑仪表盘、风险热力图

### PM视图
- 关注：任务分解、依赖关系、变更影响、团队负载
- 展示：甘特图、资源负载表、变更管理表

### 研发视图
- 关注：个人任务、代码进度、阻塞问题、技术债务
- 展示：个人任务看板、Git贡献图、阻塞问题列表

### 测试视图
- 关注：缺陷统计、质量门禁、测试进度、覆盖率
- 展示：缺陷趋势图、质量仪表盘、测试用例执行状态

## 质量门禁规则

| 阶段 | 门禁规则 | 阻塞条件 |
|------|----------|----------|
| 需求评审 | 需求文档完整度>90% | 关键需求缺失 |
| 设计评审 | 设计文档评审通过 | 架构风险未解决 |
| 代码评审 | 代码覆盖率>80%，缺陷密度<0.5/千行 | P0缺陷未关闭 |
| 测试通过 | 用例执行率100%，P0缺陷关闭 | P0/P1缺陷未关闭 |
| 上线发布 | UAT验收通过，回滚方案就绪 | 质量门禁未通过 |

## 使用示例

### 初始化项目

```bash
# 创建新项目
python rpm-fullstack init \
  --name "XX系统开发" \
  --template agile \
  --start 2026-04-06 \
  --end 2026-06-15 \
  --team team.json \
  --output ./project
```

### 更新进度

```bash
# 同步Git进度并更新任务状态
python rpm-fullstack update \
  --project ./project \
  --git-sync \
  --report daily
```

### 提交变更

```bash
# 提交变更请求
python rpm-fullstack change \
  --project ./project \
  --request change.json \
  --auto-adjust
```

### 生成复盘

```bash
# 项目结束生成复盘报告
python rpm-fullstack retro \
  --project ./project \
  --output ./report
```

## 最佳实践

1. **每日同步**：建议每天下班前更新任务进度，保持数据实时性
2. **周会回顾**：每周基于 Skill 生成的报告进行项目回顾
3. **变更审批**：所有变更必须经过审批流程，避免随意调整计划
4. **风险前置**：关注黄色预警，防止升级为红色风险
5. **复盘沉淀**：项目结束后务必生成复盘报告，积累组织经验

## 扩展开发

如需扩展 Skill 功能：

1. 在 `core/` 目录添加新的引擎模块
2. 在 `adapters/` 目录添加新的系统集成适配器
3. 在 `templates/` 目录添加新的报告模板
4. 在 `config/` 目录修改配置规则
