## 二、区域详细规范

### 2.1 筛选条件区

#### 2.1.1 布局规范

| 项目 | 规范 |
|------|------|
| **默认显示** | 一行 4 个筛选条件 |
| **超量处理** | 超过 4 个时，默认收起，显示"展开"按钮 |
| **排列方式** | 从左到右，从上到下 |
| **间距** | 筛选条件之间间距 16px |
| **对齐方式** | 左对齐，底部对齐查询按钮 |

#### 2.1.2 展开/收起交互

**默认状态（显示前4个）：**
```
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│ 条件1   │ │ 条件2   │ │ 条件3   │ │ 条件4   │
└────────┘ └────────┘ └────────┘ └────────┘
                           [重置] [查询] [展开 ▼]
```

**展开状态（显示全部）：**
```
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│ 条件1   │ │ 条件2   │ │ 条件3   │ │ 条件4   │
└────────┘ └────────┘ └────────┘ └────────┘
┌────────┐ ┌────────┐ ┌────────┐
│ 条件5   │ │ 条件6   │ │ 条件7   │
└────────┘ └────────┘ └────────┘
                           [重置] [查询] [收起 ▲]
```

#### 2.1.3 筛选条件组件

| 数据类型 | 组件 | 示例 |
|----------|------|------|
| 文本 | Input 输入框 | 课程名称、编号 |
| 枚举 | Select 选择器 | 状态、类型、院系 |
| 日期 | DatePicker 日期选择 | 创建时间、生效日期 |
| 日期范围 | RangePicker 日期范围 | 起止时间 |
| 数值 | InputNumber 数字输入 | 金额、数量 |

#### 2.1.4 操作按钮

| 按钮 | 类型 | 位置 | 说明 |
|------|------|------|------|
| 重置 | 默认按钮 | 右下角 | 清空所有筛选条件 |
| 查询 | 主按钮 | 右下角 | 触发数据查询 |
| 展开/收起 | 文字按钮 | 最右侧 | 控制筛选条件显示数量 |

### 2.2 功能点区

#### 2.2.1 位置规范

- **位置**：表格左上方
- **间距**：与筛选区间距 16px，与表格间距 16px

#### 2.2.2 默认功能点

| 功能 | 按钮类型 | 颜色 | 说明 |
|------|----------|------|------|------|
| 新增 | 主按钮 | 主题主色 | 打开新增对话框 |

| 批量导入 | 默认按钮 | 灰色 | 打开导入对话框 |
| 批量导出 | 默认按钮 | 📥 | 灰色 | 直接导出数据 |

#### 2.2.3 按钮顺序

```
[新增] [批量删除] [下载模板] [批量导入] [批量导出]
  ↑       ↑          ↑          ↑          ↑

```

#### 2.2.4 按钮状态

| 功能 | 默认状态 | 选中行后 | 说明 |
|------|----------|----------|------|
| 新增 | 可用 | 可用 | 始终可用 |
| 批量删除 | 禁用 | 可用 | 需选中数据才能操作 |
| 下载模板 | 可用 | 可用 | 始终可用 |
| 批量导入 | 可用 | 可用 | 始终可用 |
| 批量导出 | 可用 | 可用 | 始终可用 |

### 2.3 表格区

#### 2.3.1 表格结构

```
┌─────────────────────────────────────────────────────────────────┐
│  [全选]   字段1      字段2      字段3      字段4      操作栏      │ ← 表头
├─────────────────────────────────────────────────────────────────┤
│  [  ✓  ]  数据1     数据2      数据3      数据4    [查看][编辑][删除]│ ← 数据行1
│  [     ]  数据1     数据2      数据3      数据4    [查看][编辑][删除]│ ← 数据行2
│  [     ]  数据1     数据2      数据3      数据4    [查看][编辑][删除]│ ← 数据行3
└─────────────────────────────────────────────────────────────────┘
```

#### 2.3.2 表格组成

| 组成部分 | 说明 | 必显 |
|----------|------|------|
| 复选框列 | 用于批量选择 | 是 |
| 数据字段 | 业务数据展示 | 是 |
| 操作栏 | 单行操作按钮 | 是 |
| 分页器 | 数据分页导航 | 是 |

#### 2.3.3 字段规范

| 属性 | 规范 |
|------|------|
| **表头背景** | #FAFAFA |
| **表头文字** | 14px，加粗，#262626 |
| **行高** | 48-56px |
| **行分割线** | 1px solid #F0F0F0 |
| **文字对齐** | 文本左对齐，数字右对齐，操作居右 |
| **空数据** | 显示 "-" 或 "暂无数据" |

#### 2.3.4 操作栏规范

| 操作 | 样式 | 交互形式 | 说明 |
|------|------|----------|------|
| 查看 | 蓝色文字链接 | 右侧抽屉 | 查看详情，信息展示为主 |
| 编辑 | 蓝色文字链接 | 对话框 | 编辑数据，表单操作 |
| 删除 | 红色文字链接 | 对话框 | 删除确认，二次确认 |
| 更多 | 灰色文字链接/图标 | Dropdown下拉菜单 | 收起次要操作 |

**操作栏按钮顺序：**
```
[查看] [编辑] [更多 ▼]
           └── [启用]
           └── [禁用]
           └── [导出]
           └── [删除]
```

**超量处理规则：**
- **≤3个操作**：直接平铺显示所有操作按钮
- **>3个操作**：显示前2个主要操作 + 「更多」下拉菜单，次要操作收起到下拉菜单中
- **主要操作**：查看、编辑（最常用）
- **次要操作**：启用、禁用、导出、复制、删除等

**Dropdown菜单规范：**
- 菜单项按操作频率排序，高频操作在上
- 危险操作（如删除）置于菜单底部，添加分隔线
- 菜单项支持图标+文字形式
- 点击菜单项后自动关闭菜单

#### 2.3.5 分页规范

| 属性 | 规范 |
|------|------|
| **位置** | 表格下方，右对齐 |
| **组成** | 总条数 + 页码 + 每页条数选择器 |
| **默认每页** | 10 条/20 条/50 条/100 条 |
| **总条数显示** | "共 X 条" |

## 三、对话框/抽屉规范

### 3.1 对话框（Modal）规范

#### 3.1.1 使用场景

| 功能 | 使用对话框 | 原因 |
|------|-----------|------|
| 新增 | ✅ 对话框 | 表单录入，需要用户专注填写 |
| 编辑 | ✅ 对话框 | 表单编辑，需要用户专注修改 |
| 删除 | ✅ 对话框 | 需要二次确认，防止误操作 |
| 批量删除 | ✅ 对话框 | 需要二次确认，提示影响范围 |
| 批量导入 | ✅ 对话框 | 文件上传，需要展示上传进度/结果 |

#### 3.1.2 对话框结构

```
┌─────────────────────────────────────────┐
│  标题                        [X] 关闭   │ ← 头部
├─────────────────────────────────────────┤
│                                         │
│              表单内容区                  │ ← 内容区
│                                         │
│    字段1: [                    ]        │
│    字段2: [                    ]        │
│    字段3: [                    ]        │
│                                         │
├─────────────────────────────────────────┤
│           [取消]      [确定]            │ ← 底部操作区
└─────────────────────────────────────────┘
```

#### 3.1.3 对话框尺寸

| 类型 | 宽度 | 说明 |
|------|------|------|
| 小型对话框 | 400px | 删除确认、简单提示 |
| 中型对话框 | 560px | 标准表单（3-6个字段） |
| 大型对话框 | 720px | 复杂表单（6个以上字段） |

#### 3.1.4 对话框规范

| 属性 | 规范 |
|------|------|
| **头部** | 标题左对齐，关闭按钮右上角 |
| **内容区** | 表单垂直排列，字段间距 16px |
| **底部** | 左取消、右确定，按钮间距 8px |
| **遮罩层** | 可点击关闭（确认类对话框除外） |
| **动画** | 从中心缩放淡入 |

#### 3.1.5 新增/编辑对话框

## 四、交互规范

### 4.1 筛选条件交互

| 操作 | 交互效果 |
|------|----------|
| 输入筛选值 | 实时校验，回车或点击查询触发 |
| 点击查询 | 表格数据刷新，显示 Loading |
| 点击重置 | 清空所有筛选值，恢复默认状态 |
| 点击展开 | 显示全部筛选条件，按钮变为"收起" |
| 点击收起 | 只显示前3个筛选条件 |

### 4.2 功能点交互

| 操作 | 交互效果 |
|------|----------|
| 点击新增 | 打开新增对话框（Modal） |
| 点击批量删除 | 打开删除确认对话框（Modal） |
| 点击下载模板 | 直接下载模板文件 |
| 点击批量导入 | 打开导入对话框（Modal） |
| 点击批量导出 | 直接导出数据 |

### 4.3 表格交互

| 操作 | 交互效果 |
|------|----------|
| 点击全选 | 选中/取消选中当前页所有行 |
| 点击行复选框 | 选中/取消选中当前行 |
| 点击"查看" | 打开右侧抽屉（Drawer），展示详情 |
| 点击"编辑" | 打开编辑对话框（Modal） |
| 点击"删除" | 打开删除确认对话框（Modal） |
| 切换分页 | 加载对应页数据 |

### 4.4 对话框交互

| 操作 | 交互效果 |
|------|----------|
| 点击确定 | 表单校验 → 提交数据 → 关闭对话框 → 刷新表格 |
| 点击取消 | 关闭对话框，不保存数据 |
| 点击关闭图标 | 同取消 |
| 点击遮罩层 | 同取消（确认类除外） |

### 4.5 抽屉交互

| 操作 | 交互效果 |
|------|----------|
| 点击编辑 | 关闭抽屉 → 打开编辑对话框 |
| 点击关闭 | 关闭抽屉，返回列表页 |
| 点击关闭图标 | 同关闭 |
| 点击遮罩层 | 同关闭 |

## 六、示例代码结构（React + Ant Design）

```jsx
// 页面结构示例
const TablePage = () => {
  // 状态定义
  const [filterExpanded, setFilterExpanded] = useState(false);
  const [modalVisible, setModalVisible] = useState(false);
  const [drawerVisible, setDrawerVisible] = useState(false);
  const [modalType, setModalType] = useState(''); // 'add' | 'edit' | 'delete'
  const [currentRecord, setCurrentRecord] = useState(null);

  // 打开新增对话框
  const handleAdd = () => {
    setModalType('add');
    setCurrentRecord(null);
    setModalVisible(true);
  };

  // 打开编辑对话框
  const handleEdit = (record) => {
    setModalType('edit');
    setCurrentRecord(record);
    setModalVisible(true);
  };

  // 打开删除确认对话框
  const handleDelete = (record) => {
    setModalType('delete');
    setCurrentRecord(record);
    setModalVisible(true);
  };

  // 打开详情抽屉
  const handleView = (record) => {
    setCurrentRecord(record);
    setDrawerVisible(true);
  };

  return (
    <div className="table-page">
      {/* 1. 筛选条件区 */}
      <FilterSection>
        <Row gutter={16}>
          <Col span={6}><FilterItem1 /></Col>
          <Col span={6}><FilterItem2 /></Col>
          <Col span={6}><FilterItem3 /></Col>
          <Col span={6}><FilterItem4 /></Col>
          {filterExpanded && (
            <>
              <Col span={6}><FilterItem5 /></Col>
              <Col span={6}><FilterItem6 /></Col>
              <Col span={6}><FilterItem7 /></Col>
            </>
          )}
        </Row>
        <div className="filter-actions">
          <Button onClick={handleReset}>重置</Button>
          <Button type="primary" onClick={handleSearch}>查询</Button>
          <Button type="link" onClick={() => setFilterExpanded(!filterExpanded)}>
            {filterExpanded ? '收起 ▲' : '展开 ▼'}
          </Button>
        </div>
      </FilterSection>

      {/* 2. 功能点区 */}
      <ActionBar>
        <Button type="primary" icon={<PlusOutlined />} onClick={handleAdd}>
          新增
        </Button>
        <Button danger icon={<DeleteOutlined />}>批量删除</Button>
        <Button icon={<DownloadOutlined />}>下载模板</Button>
        <Button icon={<UploadOutlined />}>批量导入</Button>
        <Button icon={<ExportOutlined />}>批量导出</Button>
      </ActionBar>

      {/* 3. 表格区 */}
      <DataTable>
        <Table 
          rowSelection={{ type: 'checkbox' }}
          columns={[
            { title: '字段1', dataIndex: 'field1' },
            { title: '字段2', dataIndex: 'field2' },
            { 
              title: '操作', 
              render: (_, record) => (
                <Space>
                  <a onClick={() => handleView(record)}>查看</a>
                  <a onClick={() => handleEdit(record)}>编辑</a>
                  <a style={{ color: '#FF4D4F' }} onClick={() => handleDelete(record)}>
                    删除
                  </a>
                </Space>
              )
            }
          ]}
          dataSource={data}
          pagination={false}
        />
        <Pagination total={total} showSizeChanger showTotal />
      </DataTable>

      {/* 4. 新增/编辑对话框 */}
      <Modal
        title={modalType === 'add' ? '新增' : '编辑'}
        open={modalVisible && ['add', 'edit'].includes(modalType)}
        onOk={handleModalSubmit}
        onCancel={() => setModalVisible(false)}
        width={560}
      >
        <Form>
          <Form.Item label="字段1" name="field1" rules={[{ required: true }]}>
            <Input />
          </Form.Item>
          <Form.Item label="字段2" name="field2">
            <Select />
          </Form.Item>
        </Form>
      </Modal>

      {/* 5. 删除确认对话框 */}
      <Modal
        title="确认删除"
        open={modalVisible && modalType === 'delete'}
        onOk={handleDeleteConfirm}
        onCancel={() => setModalVisible(false)}
        width={400}
      >
        <p>确定要删除该数据吗？删除后不可恢复。</p>
      </Modal>

      {/* 6. 详情抽屉 */}
      <Drawer
        title="详情"
        placement="right"
        open={drawerVisible}
        onClose={() => setDrawerVisible(false)}
        width={640}
        footer={
          <Space>
            <Button onClick={() => {
              setDrawerVisible(false);
              handleEdit(currentRecord);
            }}>编辑</Button>
            <Button onClick={() => setDrawerVisible(false)}>关闭</Button>
          </Space>
        }
      >
        <DetailContent record={currentRecord} />
      </Drawer>
    </div>
  );
};
```
