## 1. 需求背景&目标

### 1.1 业务需求描述

地大智慧工厂教学管理平台是一个面向高校智能制造专业的综合性教学管理系统，旨在通过数字化手段提升教学管理效率，实现教务管理、教学资源管理、实训任务管理、考试管理、生产管理等业务的一体化管理。

### 1.2 用户痛点

- **教学资源分散**：视频教材、辅导教材、题库等资源分散存储，难以统一管理和共享
- **排课效率低**：手工排课工作量大，容易出现时间冲突
- **实训管理困难**：实训任务涉及设计、编程、工艺、加工、检测等多个环节，缺乏系统化管理
- **考试批阅繁琐**：传统纸质考试批阅效率低，缺乏自动批阅能力
- **刀具管理混乱**：实训过程中的刀具领用、归还、保养缺乏有效跟踪
- **数据孤岛**：各业务模块数据不互通，信息无法实时同步

### 1.3 业务目标

- **核心目标**：建立统一的教学资源管理平台，实现教学全流程数字化管理
- **业务收益**：
  - 教学资源利用率提升50%
  - 排课效率提升80%
  - 批阅效率提升60%（客观题自动批阅）
  - 刀具管理准确率达到99%

- 设备OEE、设备状态分析、设备稼动率监控、设备报警分析、报警分类表、综合报表、设备数据采集看板

### 2.2 用户画像（角色&使用场景）

#### 教务员
- **用户角色**：系统管理员
- **用户特征**：高校教务管理人员，熟悉教学管理流程，具备基础计算机操作能力
- **使用场景**：学期初维护基础数据、进行课程排课、查看教学资源
- **核心诉求**：高效管理教师、学生、课程等基础数据，合理安排课程

#### 教师
- **用户角色**：任课教师
- **用户特征**：高校专业教师，具备专业知识和教学经验
- **使用场景**：维护教学资源、发布任务/试卷、批阅作业、管理刀具需求
- **核心诉求**：便捷发布教学任务，高效批阅学生作业

#### 学生
- **用户角色**：学习者
- **用户特征**：高校在校学生，熟悉互联网产品使用
- **使用场景**：查看课程、完成任务、参加考试、管理刀具
- **核心诉求**：清晰获取任务，便捷完成任务，及时查看成绩

### 2.3 使用场景

**场景1：课程排课**
- 场景描述：学期初教务员需要为各专业课程安排任课教师、上课时间、上课地点
- 用户目标：高效完成课程排课，避免时间冲突
- 当前痛点：手工排课工作量大，容易出现时间冲突
- 解决方案：提供可视化排课界面，自动检测冲突

**场景2：普通任务发布与完成**
- 场景描述：教师需要发布课后作业，学生在线完成
- 用户目标：教师便捷发布，学生方便完成
- 当前痛点：传统纸质作业收集困难，批阅效率低
- 解决方案：在线发布任务，客观题自动批阅

**场景3：实训任务全流程**
- 场景描述：学生进行设计、编程、工艺、加工、检测等实训任务
- 用户目标：完整记录实训过程，获取实训数据
- 当前痛点：实训过程缺乏系统化管理
- 解决方案：提供五类实训任务的完整流程管理

**场景4：综合考试**
- 场景描述：教师需要组织综合考试，包含普通题目和实训题目，学生在线完成考试
- 用户目标：教师便捷组卷发布，学生公平考试，系统自动+人工批阅结合
- 当前痛点：综合考试组卷复杂，批阅工作量大，成绩统计困难
- 解决方案：支持从普通题目库和实训题目库选题组卷，客观题自动批阅，主观题教师批阅，自动生成成绩报告

**场景5：刀具管理**
- 场景描述：加工类实训需要领用、使用、归还刀具
- 用户目标：准确管理刀具流转
- 当前痛点：刀具领用归还缺乏有效跟踪
- 解决方案：与AGV对接的刀具全生命周期管理

**场景6：项目管理**
- 场景描述：教师需要创建和管理项目，设定项目阶段和任务，学生参与项目执行并跟踪进度
- 用户目标：教师便捷管理项目进度，学生清晰了解任务分工，实时跟踪项目执行情况
- 当前痛点：项目进度难以可视化跟踪，任务分工不明确，学生执行进度无法实时掌握
- 解决方案：支持项目模版管理、甘特图可视化展示项目进度、自动/手动创建项目任务、学生填写执行结果并更新任务状态

**场景7：APS智能排产**
- 场景描述：加工类任务发布后，系统根据设备状态、任务优先级、物料库存等约束条件进行智能排产，自动分配设备和生成刀具需求计划
- 用户目标：教师高效完成生产工单排产，优化设备利用率，确保任务按时完成
- 当前痛点：手工排产效率低，难以考虑多维度约束条件，设备利用率不高，排产冲突难以及时发现
- 解决方案：支持APS智能排产算法，自动检查物料库存、设备可用性等约束条件，支持优先级设置（P1-P5），支持设备更换和局部优化，排产后自动生成刀具需求计划和备刀任务单

**字段数据源：**
- 工单数据：加工类任务发布后自动生成
- 设备数据：设备管理页维护的数据
- 排产策略：排产策略管理页维护的数据

**业务逻辑：**
1. 加工类实训任务/综合试卷发布后自动生成生产工单
2. **APS排产流程（必须先设定优先级）**：
   - **步骤1-设置优先级**：教师必须先为生产工单设置优先级（P1/P2/P3/P4/P5，P1为最高优先级）
   - **步骤2-选择工单**：仅可选择已设置优先级的待排产工单进行APS智能排产
   - **步骤3-触发APS**：选择工单后触发APS智能排产算法
3. APS智能排产核心逻辑：
   - **约束条件检查（硬性约束，必须满足）**：
     - 设备约束：设备状态（在线/离线）、设备类型与工序匹配、设备当前负载
     - 时间约束：任务截止时间（即学生上课时间）
     - 物料约束（自动检查）：
       - 原材料库存充足：检查产品BOM所需的原材料库存是否满足生产需求
       - 任一物料不满足，标记排产失败并提示具体缺失物料
     - 优先级约束：P1优先级任务优先分配最优设备，按P1→P5顺序分配
   - **排产策略应用**：
     - 设备效率优先：高效率设备优先分配
     - 负载均衡优先：低负载设备优先分配
     - 交付时间优先：紧急任务优先分配
   - **最优方案计算**：综合考虑约束条件和策略，计算最优排产方案
   - **设备分配**：为每个工单分配最优设备和计划时间
4. 排产成功后自动生成备刀任务单，触发刀具准备流程
5. 排产失败时标记失败原因（无可用设备/物料不足/时间冲突）
6. 已排产的工单禁止取消发布任务
7. 支持更换已分配设备（重新触发APS局部优化）

  - 批量选择工单设置优先级（P1/P2/P3/P4/P5，P1为最高优先级）
  - 仅可选择待排产状态的工单进行设置
  - 优先级影响APS排产顺序和资源分配

| 多选题答案 | 选择 | 是 | 选择多个选项 | 请选择答案 |
| 填空题答案 | 字符串 | 是 | 非空 | 请填写答案 |
| 简答题答案 | 字符串 | 是 | 非空 | 请作答 |

**字段数据源：**
- 任务数据：教师端普通任务发布的数据
- 题目数据：普通题目页维护的数据

**业务逻辑：**
1. 教师发布后自动同步获取任务
2. 取消发布后自动删除任务
3. 支持暂存答案，稍后提交
4. 提交后等待教师批阅

**状态流转：**
| 当前状态 | 操作 | 下一状态 | 条件 |
|----------|------|----------|------|

| 已提交 | 批阅完成 | 已完成 | 教师批阅 |

- 可见角色：学生
- 可操作角色：学生

**查看页面内容描述：**
| 内容区域 | 说明 |

| 批阅结果 | 每题得分、总分、评语、正确答案、解析（仅已批阅任务显示） |

**异常处理：**
- 未答完提交：提示"还有题目未作答，确认提交？"
- 网络异常：提示"保存失败，请重试"
- 任务已取消：提示"该任务已取消"

**埋点设计：**
- 开始任务：event_stutask_start
- 暂存答案：event_stutask_save
- 提交任务：event_stutask_submit


5. 系统根据项目任务数据自动生成甘特图
6. 教师通过甘特图查看项目进度
