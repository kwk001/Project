# 生产工单详情页设计总结

## 一、页面形式

### 1.1 二级页面形式

采用**二级页面（Secondary Page）**形式，而非弹窗（Modal）形式：

```html
<div id="orderDetailPage" style="display: none; position: fixed; top: 0; left: 220px; right: 0; height: 100%; background: #fff; z-index: 2000; overflow-y: auto;">
```

**关键特点：**
- `position: fixed` - 固定定位，覆盖整个视口
- `left: 220px` - 保留左侧导航菜单（与侧边栏宽度一致）
- `right: 0` - 右侧填满剩余空间
- `z-index: 2000` - 确保在顶层显示
- `overflow-y: auto` - 内容超出时可滚动

**与弹窗形式的对比：**

| 特性 | 二级页面 | 弹窗 |
|------|----------|------|
| 视觉层级 | 独立页面，沉浸感强 | 浮层，背景可见 |
| 导航体验 | 可保留面包屑导航 | 通常只有标题栏 |
| 内容容量 | 适合大量信息展示 | 适合少量信息确认 |
| 返回方式 | 返回按钮/面包屑 | 关闭按钮/点击遮罩 |

### 2.2 模块间距设计

**间距规范：**
- 模块之间间距：`10px`
- 模块背景色：`#F5F5F5`（浅灰色）
- 模块内容背景：`#fff`（白色）
- 模块圆角：`8px`

```css
/* 页面内容区域 */
background: #F5F5F5;

/* 每个模块 */
margin-bottom: 10px;
background: #fff;
border-radius: 8px;
```

**布局要点：**

- 每个字段包含：标签（灰色小字）+ 值（黑色大字）

## 四、字段设计经验

### 4.1 字段命名规范

| 原字段名 | 优化后 | 说明 |
|----------|--------|------|
| 领料量 | 计划生产量 | 更符合生产场景 |
| 领料人 | 学生 | 明确身份 |
| 高优先级 | P1 | 使用优先级代号，简洁明了 |
| 中优先级 | P2 | 统一优先级标识体系 |

### 4.2 字段默认值

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 计划生产量 | 1 | 最小生产单位 |
| 单位 | 件 | 通用单位 |
| 实训试卷名称 | 轴类零件加工实训 | 根据业务场景预设 |

### 4.3 字段排列顺序

**原则：**
1. 核心业务信息在前（实训试卷名称、学生）
2. 物料信息紧随其后（物料名称、规格型号）
3. 生产参数随后（单位、计划生产量）
4. 归属信息在后（所属专业、所属课程）
5. 技术参数最后（BOM、工艺、起始时间段）

```javascript
// 打开详情页
function viewDetail(id) {
    const order = orders.find(o => o.id === id);
```

## 六、设计原则总结

### 6.1 视觉层次

1. **信息优先级**：核心信息（工单号、状态）> 重要信息（学生、物料）> 辅助信息（BOM、工艺）
2. **字体大小**：标题（18px）> 值（14px）> 标签（12px）
3. **颜色对比**：深色文字（#333）> 灰色标签（#999）> 背景色（#F5F5F5）

### 6.2 空间布局

1. **模块间距统一**：所有模块间距保持 `10px`
2. **内边距统一**：模块内边距保持 `16px`
3. **圆角统一**：所有模块圆角保持 `8px`

### 6.3 一致性

1. **与一级页面一致**：面包屑导航、顶部栏样式与列表页保持一致
2. **标签体系一致**：优先级标签（P1/P2）在列表页和详情页保持一致
3. **色彩体系一致**：状态颜色（红/橙/蓝/绿）保持一致

## 八、注意事项

1. **侧边栏宽度**：`left: 220px` 需要与侧边栏实际宽度保持一致
2. **z-index层级**：二级页面 `z-index: 2000` 需要高于其他元素
3. **滚动处理**：打开二级页面时禁止背景滚动，关闭时恢复
4. **响应式**：`max-width: 1200px` 限制最大宽度，保证大屏显示效果
5. **空值处理**：使用 `|| '默认值'` 确保字段为空时显示默认值
